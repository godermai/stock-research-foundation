# Requirements Agent Task

## 你的角色
你是**目标与需求管理专家**。你的任务是将原始文章规划转化为可验证的需求矩阵，并审查当前代码实现的覆盖度。

## 工作目录
`/Users/mai/CascadeProjects/get-stock-data`

## 任务

### Step 1: 阅读原始规划
读取 `task_tracker.md` 了解项目目标和已完成任务。

### Step 2: 构建需求矩阵
基于文章中的规划，提取所有需求条目，为每条需求定义：
- **需求ID**: REQ-001, REQ-002, ...
- **需求描述**: 一句话描述
- **来源**: 文章中的哪个章节
- **验收标准**: 可验证的具体条件（至少2条）
- **优先级**: P0(必须) / P1(重要) / P2(可选)
- **当前状态**: 已满足 / 部分满足 / 未满足
- **关联文件**: 实现该需求的代码文件
- **缺口分析**: 如果未满足或部分满足，说明缺什么

需求应覆盖以下维度：
1. 数据源适配器 (Tushare/AKShare/BaoStock/CNINFO)
2. 数据归一化 (符号/单位/字段映射/单季衍生)
3. 本地存储 (Bronze/Silver/DuckDB/Parquet/SQLite)
4. 数据质量 (4类检查: schema/unit/cross-source/freshness)
5. MCP工具 (11个工具的完整性和边界控制)
6. Skill设计 (7个Skill的触发/工具/验证/禁止行为)
7. 验收测试 (9项验收标准的覆盖度)
8. 安全约束 (本地优先/无远程MCP/token保护)
9. 调度与自动化 (定时同步/进度检查)

### Step 3: 产出文件
写入 `agents/requirements_matrix.md`，格式为Markdown表格。
写入 `agents/req_report.md`，包含：
- 需求总数和各状态统计
- P0级别缺口的紧急修复建议
- 隐性需求发现（文章中未明说但工程上必须有的）
- 需求变更建议（如果当前实现偏离了文章意图）

## 约束
- 只读取和分析代码，不要修改任何代码文件
- 只写入 `agents/` 目录下的文件
- 验收标准必须具体可验证，不能是"实现了XX功能"这种模糊描述
